# TAB.co.nz Odds Scraper Prompt

You are scraping football (soccer) betting odds from the open tab.co.nz tab. The output feeds an automated upload, so the JSON format must match the schema below exactly.

## Goal

For each match in the INPUT list, open its match page on tab.co.nz and capture **every market and every selection with its odds**. Return one JSON object with the same shape as INPUT, where each match object is preserved verbatim and a new `odds` object is appended to it.

## Navigation method (do this for every match)

1. From the tab.co.nz page, click into the match to open its full markets page. Do not rely on the odds shown on the match-list overview; the full market set only appears on the match page.
2. TAB groups markets under category tabs/filters (for example: Popular, Match, Goals, Goalscorers, Halves, Corners, Player, Specials). Visit every category so no markets are skipped.
3. Within each category, markets are shown as accordion panels. Many are collapsed by default. **Expand every collapsed panel** before reading it, otherwise its selections will be missing.
4. Some markets have a "Show more" / "See all" control (common on Anytime Goalscorer, Correct Score, Shots markets). Click it so all selections are visible before capturing.
5. Read every visible market and selection. Only after you have expanded and captured a page fully do you move to the next match.

## Output schema

Return a single JSON object. Keep the top-level `gamblor_round`, `dates`, `match_count`, and `matches` exactly as given in INPUT. For each match, copy every existing field unchanged and add an `odds` object:

```json
{
  "gamblor_round": 7,

  "match_count": 6,
  "matches": [

      "round": "Round of 32",
      "group": null,
```

### Field rules

- `source`: always the string `"tab.co.nz"`.
- `scraped_at`: a single ISO 8601 UTC timestamp with milliseconds and a `Z` suffix, generated once at the start of the run and reused for every match (for example `"2026-07-02T00:50:02.433Z"`).
- `market_count`: the integer length of that match's `markets` array. It must equal the actual number of market objects you output. Count them, do not estimate.
- `market`: the market's display name copied verbatim, including team names and lines (for example `"First Half Handicap Spain -1.0 Goal"`, `"Colombia Total Corners 5.5"`). Do not rename, shorten, or clean up.
- `selection`: the selection label copied verbatim (for example `"Over 2.5"`, `"Spain 5-4"`, `"Mikel Oyarzabal OR Lamine Yamal First Goalscorer"`).
- `odds`: the odds copied verbatim **as a string**, exactly as displayed. Most are two decimals (`"5.60"`). Longshots display as whole numbers and stay that way (`"101"`, `"226"`); do not add `.00`. Never convert to a number.

## Critical rules

- **Capture everything. Do not deduplicate.** TAB repeats some markets across category tabs (for example "Correct Score" and "Anytime Goalscorer" often appear more than once). Keep every instance you see. A correct scrape of these games has roughly 210 to 240 markets each.
- **Never invent, guess, or fill in odds.** If a selection has no price shown, omit that selection. If a market shows a price for only one side, include just the side that has a price (single-selection markets are valid).
- Preserve the order markets appear on the page.
- Do not translate, reformat, or "tidy" any text. Verbatim only.
- Output valid JSON only in the final deliverable: no comments, no trailing commas, correct escaping of quotes inside strings.

## Process and delivery

Work through the matches one at a time in the INPUT order. This keeps the browser session stable across the large market volume.

1. Open match 1, expand and capture all markets, then output that single completed match object as a JSON code block labelled with the match (for example `Spain vs Austria`). Include its `market_count`.
2. Repeat for each remaining match, outputting one code block per match as you finish it. Emitting each game as you go means nothing is lost if the session is interrupted.
3. After the last match, output the complete assembled JSON object (all matches inside the full top-level structure) as the final deliverable.

Before delivering each match block, self-check: does `market_count` equal the number of market objects, are all `odds` values strings, and did you keep duplicate markets rather than merging them?
